Conditionals

If statements

If statements are used to make sure that the program only does something if certain conditions are met. An example is shown below:


a = 6

b = 5

if a + b == 11

puts “Hello Spaceman”

end

Example: Since a + b = 5 + 6 = 11, “Hello Spaceman” will be printed to the console on a new line. The “end” is just there to highlight the end of the if clause


You may be wondering why two equals signs were used for the if statement. This means that the program is comparing the two values. In this instance it is comparing the result of a + b against 11.

There are several other “comparators” which can be used in Ruby:

  • == (Check if the values either side are equivalent)
  • != (Check if the values either side are unequal)
  • < (Check if the value on the left is strictly less than the value on the right)
  • <= (Check if the value on the left is less than or equal to the value on the right)
  • > (Check if the value on the left is strictly greater than the value on the right)
  • >= (Check if the value on the left is greater than or equal to the value on the right)

Else and Elsif statements

We can also provide alternate options for the program if the initial condition isn’t met using the else or elsif functions.

x = 5

if x == 5

puts "This is great!"

elsif x <= 4

puts "Nope, there's no Spaceman to say hello to!"

elsif x >= 6

puts "There are too many Spacemen, something has gone wrong!"

end

Example:elsif statements.


Since x is equal to 5, the output will be "This is great!". If the value of x is edited to something less than 5, then the 1st elsif condition will be satisfied and it will output "Nope, there's no Spaceman to say hello to!". Finally if we edited the value of x to be greater than 5, then the final elseif condition would be satisfied and it would output "There are too many Spacemen, something has gone wrong!"



x = 5

if x < 5

puts "We're almost there"

else

puts "We're here!"

end

Example: else statements. Here, since x equals 5, the condition isn't satisfied so the code goes to the "else" function and will output "We're here!"


Unless Statements

Unless statements are the opposites of if statements where the function will be perfomed unless the condition is fulfilled.


j = 3

unless j = 15

puts "j is not 15"

end

Example: We have stated that j has a value of 3. The unless statment's condition is such that "j is not 15" will be outputted so long as (shockingly) j is not 15.


Logical Operators

Logical operators are used in conditional statements where two conditions need to be satisfied, for example if value a needs to be greater than 3, but value b must be 5 exactly

There are several logical operators in Ruby:

  • && (Logical AND operator: both conditions must be satisfied )
  • and (same as &&)
  • || (Logical OR operator: at least one of the conditions must be satisfied)
  • or (same as ||)
  • ! (reverses the logic of the statement so that it's false when true and true when false)
  • not (same as !)

Here are some examples to help illustrate how these would be used:

y = 12

z = 5

if (y <= 15 || z >= 15)

puts "Blast off!"

else

puts "There's a fuel leak, we are unable to take off for now"

end

Example: The || operator is being used here ("||" could have been replaced by "or" for the same functionality"). As one of the conditions is fulfilled the output will be "Blast off!"


y = 12

z = 5

if !(y <=15 || z >= 15)

puts "There's a problem with the carburetor"

elsif (y = 12 && z == 5)

puts "There's a problem with the lunar module"

end

Example: The ! operator reverses the logic of the conditions. Thus, here the ! causes the true false to be recognised as false, thus the output cannot be "There's a problem with the carburetor". As both the conditions in the elseif statement are satisfied, the output will be "There's a problem with the lunar module".
