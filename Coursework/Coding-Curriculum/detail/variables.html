Variables

Types of variables

There are 5 types of variables you can create in Ruby: local, instance, class, global and constants.

Local variables are variables that are only accessible from within the scope they are defined such as in a function. They are indicates by starting with a lowercase letter and are typically named in camel caps which means the first letter is lower case and then the start of each new word is capitalised e.g. capitalCity.

number = 42

Instance variables are variables specific to an object of a class, they are defined in a class but each object has their own values for them. Instance variables start with an '@' symbol, once again they are typically named in camel caps.

@number = 42

Class variables are variables that are shared by all variables of a class (and child classes which will be explained later) so they are declared in the class and then any objects created of that class can access that variable. Class variables start with two '@' symbols and are normally named in camel caps.

@@number = 42

Global variables are variables that can be accessed anywhere in the program, they should be avoided as far as possible as they can make code harder to understand. Global variables start with a '$' symbol and normally are named in camel caps.

$number = 42

Constants are variables that cannot change so the value they are created with will always be their value. Constants start with an upper case letter and it is common to name them in all capital letters, spearating words with '_'.

NUMBER = 42

Variables can store any object, and how to do that will be explained later, here however we will discuss some basic literals, which allow you to easily create certain objects


Numbers can be used just by typing the number and Ruby will create an object suitable to hold the value, they can be created like so:

life = 42

pi = 3.14159265358979

This defines a variable called life ith a value of 42 and pi with a value of 3.14159265358979

You can also create String objects to store text as a literal using speech marks or quotation marks like so:

string = "Strings are cool"

ruby = 'Ruby is cooler'

This defines two variables, string and ruby containing text as a string.

Going beyond this you can also display other values within the string using the syntax #{ expression }, however you have to use speech marks to do this, here are some examples:

number = 42

life = "The meaning of life is #{ number }"

addition = "2 plus 3 equals #{ 2 + 3}"

Here a variable called number is declared with the value 42 and the life variable uses that to create the string "The meaning of life is 42". A variable named addition is also defined that evaluates an expression to create a string "2 plus 3 equals 5".
